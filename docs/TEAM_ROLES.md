# XingJu 项目团队职责说明

## 团队架构

### 🎯 Orion（项目经理）
- **职责**：任务拆解、进度管控、风险管理、团队协调
- **工作流**：Orion → Arch → Dev → Test → Orion 验收
- **交付物**：项目计划、里程碑跟踪、风险管理

### 🏗️ Arch（架构师）
- **职责**：
  - 系统架构设计
  - 技术选型决策
  - 性能优化方案
  - 跨平台兼容性设计
  - 代码规范制定
- **在 XingJu 项目中**：
  - 设计 Rust + Tauri + React 技术栈架构
  - 制定 API 接口规范
  - 确保前后端通信协议
  - 优化跨平台构建流程
- **协作方式**：接收 Orion 任务分解 → 输出架构方案 → 交给 Dev 实施

### 💻 Dev（开发工程师）
- **职责**：
  - 功能模块开发
  - API 接口实现
  - 前后端集成
  - 代码质量保证
  - 性能优化实现
- **在 XingJu 项目中**：
  - 实现音乐/视频/小说/漫画聚合功能
  - 接入真实第三方 API
  - 保持赛博朋克 UI 风格
  - 确保跨平台兼容性
- **协作方式**：接收 Arch 架构方案 → 实施开发 → 交付 Test 验证

### 🧪 Test（测试工程师）
- **职责**：
  - 功能测试验证
  - 性能测试评估
  - 兼容性测试
  - Bug 挖掘与追踪
  - 质量保障
- **在 XingJu 项目中**：
  - 验证音乐/视频/小说/漫画功能
  - 测试跨平台兼容性
  - 验证 API 接口稳定性
  - 确保 UI 一致性
- **协作方式**：接收 Dev 完成功能 → 执行测试 → 反馈结果给 Orion

## 工作流程

```
Orion（任务分解） → Arch（架构设计） → Dev（开发实现） → Test（测试验证） → Orion（验收交付）
```
